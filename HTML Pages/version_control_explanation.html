What is Version Control?

Version control is a way to manage file systems and code bases. It allows us to incrementally improve upon code without worrying about breaking code that already functions properly. New changes can be tested separate from the code base, and once the new changes are approved the new changes can be added to the code base.

If you take a look at the slightly confusing diagram, you will see the general way that version control works. In brief, version control works like this:

  1. Each developer creates their own repository, or "copy" of the code, from the source code.
  2. The developer creates a branch to work on new features, fix bugs, etc.
  3. When the developer has finished working on the feature, bug, etc., the developer has their code reviewed.
  4. Once code has been reviewed, the branch is merged into the source.
  5. Each developer updates their repository to include the new changes, and the cycle repeats.

But Why Should I Care About Version Control?

Excellent question. Think about it this way: what if coding was like designing a car? Each year a new car model comes out. They don't reinvent the car each time they want to improve on something, that would be expensive and time consuming. Instead they incrementally improve on each car. And what if there is a factory recall for a problem? They are able to fix the cars as the problems arise without worrying about affecting other systems. To take the analogy even further, oftentimes different models of car brands use the same parts. If they reinvented the car each time they released a new one, they would be unable to do that.

Like designing cars, coding uses the same principles for incrementally improving and adding new features. Version control allows us to use, reuse, and improve upon our already existing code base without having to reinvent it every time. If we worked on the same project using the same files, eventually someone would do something that would break our carefully constructed system. Finding the issue would be much more difficult without a dedicated history of the changes we had made or without the ability to revert back to a previous version of the software. Version control allows us to to both of those things, and you will see the benefits of that as we work on this project.
